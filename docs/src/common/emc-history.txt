== Origen

EMC (Controlador de Máquina Mejorado) fue creado por
http://www.nist.gov/index.html[NIST], el Instituto Nacional de Normas
y Tecnología, que es una agencia del Departamento de Comercio del
Gobierno de los Estados Unidos.

NISTse interesó primero en escribir un paquete de control de movimiento como plataforma de prueba
de conceptos y estándares. El patrocinio de General Motors
resultó en una adaptación de la versión incipiente de EMC usando 
tarjetas de control PMAC inteligentes que se ejecutaban en una versión "en tiempo real" de Windows NT
y controlaban una gran fresadora.

Al igual que todos los 'productos de trabajo' de los empleados del gobierno federal de EE.UU.,
el software resultante y el informe al respecto deben estar en el
dominio público y un informe al respecto debidamente publicado, incluso en
Internet. Fue allí donde Matt Shaver descubrió EMC. Matt contactó con el NIST
y entró en conversaciones con Fred Proctor sobre la adaptación del código para
controlar hardware menos costoso que se utilizaria para actualizaciones y
reemplazos de controles de CNC que estaban obsoletos o simplemente muertos. NIST
estaba intrigado porque ellos también querían algo menos costoso. Con el fin de
lanzar un esfuerzo cooperativo, se creó un acuerdo formal que garantizó
que el código resultante y el diseño permanecerían en el dominio público.

Las primeras consideraciones se centraron en reemplazar el costoso y temperamental
sistema Windows NT "en tiempo real". Se propuso que se intentara con una relativamente nueva (en el
tiempo) extensión en tiempo real del sistema operativo Linux. Esta idea
fue implementada con éxito. El siguiente fue el tema del la costosa tarjeta inteligente
de control de movimiento. En aquel momento, la potencia de procesamiento de una PC era
considerado lo suficientemente grande como para tomar el control directo de las rutinas de movimiento.
Una búsqueda rápida de hardware disponible resultó en la selección de la placa de interfaz
http://www.servotogo.com/[Servo-To-Go] como la primera plataforma
para permitir que la PC controlase directamente los motores. El software para planificación de trayectoria
y control PID de bucle se agregó a la interfaz de usuario existente y al
intérprete RS274. Matt utilizó con éxito esta versión para actualizar un par
de máquinas con controles muertos y este se convirtió en el sistema EMC que primero
llamó la atención del mundo exterior. La mención de EMC en el grupo de noticias USENET
rec.crafts.metalworking produjo los primeros interesados en adoptar sistemas de construcción EMC
como http://pico-systems.com/motion.html[Jon Elson].

NIST creó una lista de correo para las personas interesadas en EMC. Con el paso del tiempo,
otros distintos del NIST se interesaron en mejorar EMC. Mucha gente solicitó
pequeñas mejoras codificadas en el código. Ray Henry quería refinar el interfaz de usuario.
Como Ray era reacio a intentar alterar el código C en el que
se escribió la interfaz de usuario, se buscó un método más simple. Fred Proctor, del
NIST, sugirió un lenguaje de script y escribió el código para interactuar con Tcl/Tk
para las comunicaciones internas NML de EMC. Con esta herramienta
Ray pasó a escribir un programa Tcl/Tk que se convirtió en el interfaz de usuario predominante
para EMC en el momento.

Para la perspectiva del NIST, vea este
http://web.archive.org/web/20120417094958/http://www.isd.mel.nist.gov/documents/shackleford/4191_05.pdf[documento]
escrito por William Shackleford y Frederick Proctor, describiendo la
historia de EMC y su transición a código abierto.

En este momento, el interés en EMC comienza a recuperarse sustancialmente. Como más
y más personas intentaron instalar EMC, la dificultad de
parchear un kernel de Linux con las extensiones en tiempo real y compilar el
código EMC se volvió notoriamente obvia. Hubo muchos intentos de documentar el proceso y
se intentaron escribir guiones, algunos con éxito moderado. El problema de
hacer coincidir la versión correcta de los parches y los compiladores con la 
la versión de Linux seleccionada siguió apareciendo. Paul Corner vino al rescate con el
BDI (instalación de cerebro muerto) que era un CD del cual podrían instalarse un sistema de trabajo completo
(Linux, parches y EMC). El enfoque BDI abrió el
mundo de EMC a una comunidad de usuarios mucho más grande. Mientras esta comunidad continuaba
creciendo, la lista de correo de EMC y los archivos de código se movieron a
http://www.sourceforge.net/projects/emc/[SourceForge] y se estableció el sitio web LinuxCNC.

Con una comunidad más grande de usuarios participando, EMC se convirtió en el foco principal de
interés en las exposiciones CNC en curso en NAMES y NAMES se convirtió en el evento anual
de reunión para EMC. Durante los primeros años, las reuniones solo
sucedian porque las partes interesadas estaban en NAMES. En 2003, la comunidad de usuarios de EMC
tuvo su primera reunión pública anunciada. Se celebró el lunes
después de NAMES en el vestíbulo de la arena donde se celebró el show NAMES.
La organización era floja, pero nació la idea de una capa de abstracción de hardware
(HAL) y  fue propuesto el movimiento para reestructurar el código para facilitar el desarrollo (EMC2).

== Cambio de nombre

En la primavera de 2011, la Junta Directiva de LinuxCNC fue contactada por
una firma de abogados que representa a EMC Corporation (www.emc.com) sobre el uso de
"EMC" y "EMC2" para identificar el software ofrecido en linuxcnc.org. EMC
Corporation ha registrado varias marcas comerciales relacionadas con EMC y EMC²
(EMC con numeral superíndice dos).

Después de varias conversaciones con el representante de EMC
Corporation, el resultado final fué que, comenzando con el siguiente gran
lanzamiento del software, linuxcnc.org dejará de identificar el software
usando "emc" o "EMC", o aquellos términos seguidos de dígitos. En la medida
que la Junta Directiva de LinuxCNC controla los nombres utilizados para identificar
el software ofrecido en linuxcnc.org, la junta ha aceptado esto.

Como resultado, fue necesario elegir un nuevo nombre para el software. De
las opciones que la junta consideró, hubo consenso de que "LinuxCNC" era
la mejor opción, ya que este ha sido el nombre de nuestro sitio web durante años.

En preparación para el nuevo nombre, hemos recibido una sublicencia de la
Marca registrada LINUX® de la Fundación Linux (www.linuxfoundation.org),
protegiendo nuestro uso del nombre LinuxCNC. (LINUX® es la marca registrada
de Linus Torvalds en los EE. UU. y otros países).

El esfuerzo de cambio de marca incluirá el sitio web linuxcnc.org, el canal IRC
y las versiones del software y la documentación que comienzan con la 2.5.0.
